# Reference Meme Styles

These are visual recipes extracted from real Chinese/tech meme references. Use them
when a user says the output needs more "网感" instead of polished poster design.

## Sticker Store Roast

Visual structure:
- Tall mobile screenshot, mostly white negative space.
- A small central mascot/sticker image.
- One red caption bar directly under or over the sticker.
- Caption is short, blunt, and slightly mean.
- The joke works because the image looks like a saved sticker, not a designed poster.

Good for:
- "老实人被暗箭所伤"
- "工具作者被真实用户暴击"
- "开源项目无人问津"

Caption patterns:
- 老实人被暗箭所伤
- 作者本人受到暴击
- 沉默，是今晚的 GitHub
- 这下真没人用了

## X Screenshot Tech Collage

Visual structure:
- Tall black X/Twitter screenshot shell.
- Fake account header, handle, follow button, top status bar.
- Main meme is a white card inserted in the timeline.
- Top of white card has large plain black text, almost PowerPoint-like.
- Bottom image is a chaotic collage: logos, characters, pasted labels, explosions, screenshots.
- Engagement footer adds social proof.

Good for:
- developer jokes
- OS/tool comparisons
- GitHub/AI agent/skill memes
- "Linux:" punchlines

Caption patterns:
- Windows 和 macOS：复杂且优雅地 X / Linux：咔咔就是干
- 普通 AI 工具：请先阅读文档 / 我的 skill：先整活
- 别人的开源项目：架构图 / 我的开源项目：求求你点 Star
- 传统宣传：写卖点 / 技术圈宣传：做成烂梗

## Ugly-Funny Rules

- Do not over-polish.
- Prefer deadpan labels over clever slogans.
- Preserve platform context when it helps the joke.
- White cards, black text, yellow sticky labels, red bars, and crude pasted assets are allowed.
- The joke should be understandable in 1 second.
- If the image looks like a tasteful product poster, it is probably wrong.
